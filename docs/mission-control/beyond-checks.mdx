title: "Beyond Checks"
---

Checks review every pull request automatically. But pull requests are just one trigger. Smart AI can also run agents on schedules, in response to events like new issues, Sentry alerts, or Slack messages.

## Agents

Agents use the same markdown file format as checks. The difference: agents can be triggered by events beyond PR open — schedules, new issues, CI failures, webhooks, and more. Triggers are configured in the [Mission Control UI](https://smart-ai.dev).

### Checks vs. agents

| | Checks | Agents |
|---|---|---|
| File location | `.smart-ai/checks/` | `.smart-ai/agents/` or smart-ai.dev |
| Trigger | Always on PR open | Configurable in Mission Control |
| Purpose | Pass/fail review of PRs | Any automated task |
