## How to Use Text Input

Typing a question or instructions into the input box is the only required context.

## How to Include Highlighted Code

The highlighted code you've selected by pressing cmd/ctrl + L (VS Code) or cmd/ctrl + J (JetBrains) will be included in your prompt.

## How to Include the Active File

You can include the currently open file as context by pressing opt/alt + enter when you send your request.

## How to Include a Specific File

You can include a specific file in your current workspace by typing '@Files' and selecting the file.

## Codebase Search

For better codebase awareness, see our [guide on making agent mode aware of codebases and documentation](/guides/codebase-documentation-awareness).

## How to Include Documentation Sites

For better documentation awareness, see our [guide on making agent mode aware of codebases and documentation](/guides/codebase-documentation-awareness).

## How to Include Terminal Contents

You can include the contents of the terminal in your IDE by typing '@Terminal'.

## How to Include Git Diff

You can include all of the changes you've made to your current branch by typing '@Git Diff'.

## How to Include Open Files

You can include all currently open files by typing `@Open Files`. This is useful when working across multiple related files.

## How to Include the Entire Codebase

For repo-wide awareness, type `@Codebase` to automatically retrieve relevant files based on your query. This requires an embeddings model (configured by default in the Ollama template).

## How to Include a Folder

You can include all files in a specific folder by typing `@Folder` and selecting the directory.

## How to Use Other Context Providers

You can see a full list of built-in context providers [here](/customize/deep-dives/custom-providers).

### Default Context Providers

Smart AI enables the following providers by default for a Copilot-style experience:

| Provider | Description |
|----------|-------------|
| `@Files` | Reference any file in the workspace |
| `@Current File` | The currently open file |
| `@Open Files` | All currently open files |
| `@Codebase` | Automatically find relevant files |
| `@Folder` | Reference an entire folder |
| `@Git Diff` | Current git diff |
| `@Terminal` | Last terminal command output |
| `@Problems` | Diagnostics / errors |
| `@Rules` | Custom rules from `.continuerules` |

## Copilot-Style Workflow

Smart AI is designed to feel like a GitHub Copilot alternative. Here is the recommended workflow:

1. **Chat** (`Cmd/Ctrl+L`) - Ask questions about your code or general programming topics
2. **Edit** (`Cmd/Ctrl+I`) - Select code and ask Smart AI to modify it inline
3. **Autocomplete** (`Tab`) - Accept fast, context-aware completions as you type
4. **Apply** - Paste code from chat into files with a diff preview
5. **Agent** - Enable agent mode for multi-step coding tasks

### Quick Context Tips

- `@Codebase` - Ask "where is X defined?" or "how do I use Y?" without manually adding files
- `@Git Diff` - "Review my changes" before committing
- `@Problems` - "Fix these type errors" using diagnostics
- `@Terminal` - "Why did this build command fail?"
- `@Folder src/components` - "Refactor all components in this directory"

Smart AI works best when you treat it as a pair programmer: describe what you want, provide context with `@` mentions, and iterate on the results.
